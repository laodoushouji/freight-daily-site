# Proposal: freight-daily-v2

## Why

当前站点（v1/v2过渡态）存在根本性问题：

1. **内容无人看** — 5篇文章全是英文航运新闻翻译，货代从业者不会每天来看这种内容
2. **数据是手填的** — briefs/2026-06-18.json 是人工编造的示例，无真实数据源
3. **没有闭环** — 发布后无反馈、无分析、无优化，是"设定好就死跑的脚本"
4. **定位模糊** — 到底是新闻站、数据站、还是决策辅助？全沾一点但都不专

**核心洞察**：货代人不会为"新闻"每天打开网站，只会为"省钱+省时间"每天打开。

**为什么现在做**：
- v2 首页模板已改为信息面板，但内容层全是空的
- 每天跑的 cron 在产出没人看的内容，浪费算力
- 竞品（搜航/锦程）体验差，有差异化空间

## What Changes

将货代日报站从"航运新闻站"彻底重建为"货代人每日决策简报"：

1. **数据层重建** — 从真实公开数据源自动抓取运价/船司/港口信息
2. **内容层重建** — 从"5篇长文"改为"简报 + 2-3篇深度分析"
3. **迭代层新建** — 搜索流量分析 → 内容策略自动调整
4. **交互层新建** — 用户反馈按钮 + 订阅推送

## Capabilities

### New Capabilities

- `data-pipeline`: 自动从公开数据源采集运价指数、船司公告、港口动态，结构化存储
- `daily-brief`: 每日生成运价风向 + 船司动态 + 港口预警 + 操作建议的决策简报
- `deep-analysis`: 基于简报数据生成2-3篇货代视角的深度分析文章
- `feedback-loop`: 收集用户反馈，分析搜索流量，自动调整内容策略
- `subscription`: 按航线/主题订阅推送（邮件或钉钉）

### Modified Capabilities

- `content-generation`: 从"翻译英文新闻"改为"数据驱动+货代视角分析"
- `site-layout`: 首页从文章列表改为信息面板（已部分完成）
- `cron-pipeline`: 从单一流程改为多阶段流程（采集→生成→审查→发布）

## Impact

### 代码影响
- `scripts/fetch_news.py` → 重写为多数据源采集器
- `scripts/build_site.py` → 重写为支持简报+文章双模式
- `scripts/generate_content.py` → 重写为数据驱动内容生成
- `templates/*` → 大部分需要更新
- `briefs/` → 改为程序生成而非手填

### 数据依赖
- SCFI 指数（上海航运交易所，公开）
- Freightos 运价指数（公开 API）
- 船司官网公告（需适配解析）
- 港口等泊数据（MarineTraffic / 港口官网）

### 运营影响
- Cron 从 1 次/天 可能增至 2-3 次/天（采集+生成+审查）
- 模型调用量增加（但仍在免费额度内）
- Vercel 部署频率增加（每天 1-2 次）

### 回滚方案
- 所有改动在 freight-daily-v2 分支开发
- 主分支保持当前 v1 状态
- 新数据源采集器独立运行，不影响现有内容
- 随时可切回 v1 内容模式
