# Design: freight-daily-v2

## Context

货代日报站 v1 已上线 Vercel，当前是 v1→v2 过渡态：
- 首页模板已改为信息面板（运价+船司+港口+操作建议）
- 但数据层全是手填的 JSON
- 内容生成逻辑仍是翻译英文新闻
- 无数据采集、无反馈、无迭代

需要从数据层到呈现层全量重建。

## Goals / Non-Goals

**Goals:**
- 货代人每天早上 5 分钟能看到"今天该注意什么"
- 数据从公开源自动采集，零人工干预
- 站点自我迭代：标题策略进化、内容比例调整、搜索收录优化
- 月运营成本 ¥0（硬约束）

**Non-Goals:**
- 不做运价查询工具（搜航/锦程已做）
- 不做订舱平台（太重）
- 不做实时追踪（技术栈不支持）
- v2.0 不做按航线订阅（放 v2.1）
- v2.0 不做国内备案站（放后续）

## Decisions

### D1: 数据采集策略 — 80%自动 + 20%模板

**决策**：不是所有数据都能 100% 自动采集。船司公告格式不统一，港口中文数据难抓。

**方案**：
- SCFI/Freightos → 全自动（公开网页抓取）
- 船司公告 → AI 辅助提取（抓取公告页 → AI 提取结构化信息）
- 港口动态 → 英文源为主（MarineTraffic），中文源半自动
- 采集失败 → 保留上次数据 + 标注"暂未更新"

### D2: 内容生成 — 简报优先，文章为辅

**决策**：首页核心是简报面板，不是文章列表。文章是简报的延伸。

**方案**：
```
每日流程：
  1. 采集数据 (05:00 cron)
  2. 生成简报 JSON (AI 基于数据)
  3. 选 2-3 个话题写深度文章 (AI 基于简报)
  4. 构建站点 (Jinja2)
  5. git push → Vercel 部署
  6. 钉钉推送摘要
```

### D6: 免费数据源确认

| 数据 | 来源 | 成本 | 可靠性 | 更新频率 |
|------|------|------|--------|---------|
| SCFI 指数 | 上海航运交易所官网 | 免费 | 高 | 每周五 |

## Risks / Trade-offs

### R1: 数据源不稳定
- **风险**：SCFI 网站改版、船司官网改版导致采集失败
- **缓解**：多数据源互备 + 采集失败降级策略 + 采集器版本化

### R2: AI 生成内容质量
- **风险**：免费模型可能生成不准确的数据解读
- **缓解**：数据层和解读层分离（数据是采集的，不是 AI 编的）；操作建议基于规则模板而非纯 AI

### R3: SEO 冷启动
- **风险**：前 30 天搜索引擎几乎不收录
- **缓解**：提交 sitemap 到 Google Search Console / 百度站长；标题含年份+数字提升收录率

### R4: 零成本约束
- **风险**：免费额度可能不够
- **缓解**：Vercel 免费套餐 100 次部署/天足够；免费模型 API 额度覆盖 50-80 次/天调用量

### R5: 时间投入
- **风险**：全量重建可能需要较长开发时间
- **缓解**：按 OpenSpec 任务拆解，每个任务 2-5 分钟（Superpowers writing-plans 标准）
